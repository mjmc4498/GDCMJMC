📋 Descripción del Módulo

El Modelado de Datos es el proceso de creación de representaciones abstractas de los datos y sus relaciones para comprender, documentar y comunicar las reglas y estructuras del negocio. Un buen modelo de datos es la base de una arquitectura de información sólida y la clave para sistemas mantenibles y escalables.

📘 Relación con DAMA-DMBOK

En el DAMA-DMBOK, el Modelado y Diseño de Datos es el dominio que establece los estándares, actividades y mejores prácticas para crear modelos de datos de calidad. Incluye los tres niveles de abstracción (conceptual, lógico, físico), las notaciones estándar (IE, UML, Bachman) y el proceso de revisión y aprobación de modelos.

⚖️ Relación con Gobierno Corporativo

Desde el gobierno corporativo, los modelos de datos son activos empresariales que requieren control de versiones, aprobación formal y alineación con la estrategia de información. La alta dirección necesita entender las entidades de negocio clave (Cliente, Producto, Transacción) que están modeladas, ya que decisiones estratégicas dependen de la calidad de estos modelos.

🔒 Cumplimiento — Ley N.° 29733

El Reglamento de la Ley 29733 requiere que las organizaciones documenten las finalidades del tratamiento de datos personales. El Modelo de Datos es el instrumento técnico que demuestra qué datos personales se almacenan, en qué entidades, con qué atributos y con qué finalidad, siendo evidencia técnica ante la Autoridad Nacional de Protección de Datos Personales (ANPDP).

🎯 Objetivos de Aprendizaje

  • Diseñar modelos de datos en sus tres niveles de abstracción según el estándar DAMA-DMBOK.
  • Aplicar técnicas de normalización (1FN, 2FN, 3FN, BCNF) correctamente.
  • Utilizar notaciones estándar (IE Crow's Foot, UML) en el modelado de datos.
  • Distinguir entre modelos OLTP (transaccionales) y OLAP (analíticos).
  • Documentar las entidades de datos personales según los requisitos de la Ley 29733.

🔑 Conceptos Clave

  • Entidad: Objeto del mundo real sobre el que se almacena información: Cliente, Producto, Pedido.
  • Atributo: Propiedad o característica de una entidad: Nombre, Fecha de Nacimiento, DNI.
  • Relación: Asociación entre dos entidades: un Cliente REALIZA muchos Pedidos.
  • Normalización: Proceso de organizar los datos para reducir redundancia e inconsistencia mediante formas normales.
  • Data Vault: Metodología de modelado para Data Warehouses que separa estructura (Hubs), relaciones (Links) y atributos históricos (Satellites).

⚙️ Explicación Técnica Detallada

El proceso de modelado sigue una secuencia rigurosa: (1) Relevamiento — entrevistar al negocio para identificar entidades y reglas; (2) Modelo Conceptual — entidades y relaciones en lenguaje de negocio usando notación ER; (3) Revisión con el negocio y aprobación formal por el Data Owner; (4) Modelo Lógico — agregar atributos, claves primarias/foráneas, tipos de datos lógicos y reglas de negocio; (5) Modelo Físico — adaptar al motor de base de datos específico con índices, particionamiento y consideraciones de rendimiento; (6) Versionado y publicación en el catálogo de datos corporativo.

🏢 Ejemplos en Organizaciones

La AFP Integra en Perú rediseñó su modelo de datos de afiliados usando modelado dimensional (Star Schema) para su Data Warehouse. Esto permitió calcular el valor de fondo de pensiones de 2 millones de afiliados en minutos en lugar de horas, mejorando la experiencia del cliente y los reportes regulatorios a la SBS.

✅ Buenas Prácticas Internacionales

  • Siempre crear primero el modelo conceptual con el negocio antes de pasar al lógico.
  • Usar un estándar de nomenclatura consistente para tablas, atributos y claves.
  • Versionar todos los modelos en un repositorio con control de cambios.
  • Revisar y aprobar formalmente cada modelo con el Data Owner del dominio.
  • Documentar en el modelo los atributos que contienen datos personales para cumplir Ley 29733.

⚠️ Errores Comunes

  • Saltar directamente al modelo físico sin pasar por el lógico, generando deuda técnica.
  • No normalizar correctamente, generando duplicidad de datos e inconsistencias.
  • No documentar las reglas de negocio en el modelo, perdiendo el conocimiento organizacional.
  • Usar modelos individuales por sistema sin una vista empresarial unificada.
  • No marcar los atributos de datos personales, dificultando el cumplimiento de la Ley 29733.

💼 Casos de Uso Reales

Netflix usa modelos de datos distribuidos (Cassandra + modelos desnormalizados) para servir recomendaciones a 220 millones de usuarios. En el sector público peruano, la SUNAT migró su modelo de datos de contribuyentes a un esquema normalizado que permitió la implementación de la Factura Electrónica, reduciendo el fraude tributario en 18%.

🛠️ Herramientas y Tecnologías

  • erwin Data Modeler: Modelado conceptual, lógico y físico con ingeniería directa e inversa.
  • PowerDesigner (SAP): Suite completa de modelado con soporte a Data Vault y MDM.
  • dbdiagram.io: Herramienta web gratuita para modelado rápido con notación DSL.
  • MySQL Workbench / pgModeler: Modelado físico específico para MySQL y PostgreSQL respectivamente.
  • dbt + YAML: Documentación de modelos de datos en pipelines de transformación modernos.

✏️ Ejercicio Práctico

Actividad: Dado el enunciado de una cadena de farmacias peruana con farmacias, productos, clientes con recetas médicas y ventas, elabora: (a) Modelo Conceptual con mínimo 6 entidades; (b) Modelo Lógico normalizado en 3FN con todos los atributos y claves; (c) Identificación de atributos de datos personales sensibles (datos de salud) según Ley 29733; (d) Justificación de las decisiones de diseño en las relaciones N:M.

💡 Recomendaciones de Implementación

  • Establecer un proceso formal de revisión de modelos con el Architecture Review Board.
  • Publicar los modelos aprobados en el catálogo de datos con su documentación.
  • Usar lineaje de datos desde el modelo hasta los reportes para demostrar trazabilidad.
  • Incluir en el modelo el campo de consentimiento del titular para datos personales (Ley 29733).
  • Capacitar a los desarrolladores en lectura e interpretación de modelos de datos.

📌 Resumen del Módulo

El Modelado de Datos es la disciplina que convierte el conocimiento del negocio en estructuras de información formales y reutilizables. El DAMA-DMBOK establece un proceso riguroso de tres niveles que garantiza calidad, consistencia y alineación con el negocio. Un modelo bien construido es también la principal evidencia técnica de cumplimiento de la Ley 29733 en cuanto a documentar finalidades y estructuras de tratamiento de datos personales.
