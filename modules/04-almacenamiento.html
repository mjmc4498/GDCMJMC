📋 Descripción del Módulo

El Almacenamiento de Datos abarca el diseño, implementación y administración de las tecnologías y plataformas que persisten los datos de la organización. Incluye bases de datos relacionales, NoSQL, almacenes de datos (Data Warehouses), lagos de datos (Data Lakes) y bases de datos en la nube, así como la gestión de su rendimiento, disponibilidad y seguridad.

📘 Relación con DAMA-DMBOK

En el DAMA-DMBOK, el dominio de Almacenamiento y Operaciones de Datos cubre las actividades técnicas de gestión de plataformas de datos: administración de bases de datos (DBA), garantía de disponibilidad (SLAs), gestión de respaldos (backup/recovery), optimización de rendimiento (tuning) y planificación de capacidad. Es la infraestructura operativa sobre la que reposan todos los demás dominios.

⚖️ Relación con Gobierno Corporativo

Desde el gobierno corporativo, las decisiones sobre tecnologías de almacenamiento son inversiones estratégicas que el directorio supervisa. El tiempo de inactividad de una base de datos crítica tiene impacto financiero directo. Las políticas de retención y eliminación de datos son también decisiones de gobierno que vinculan a los responsables legales de la organización con la infraestructura técnica.

🔒 Cumplimiento — Ley N.° 29733

La Ley 29733 en su artículo 9 establece el principio de seguridad, exigiendo que los titulares de bancos de datos personales implementen medidas de seguridad técnicas y organizativas. En el almacenamiento, esto implica: cifrado de datos en reposo, control de acceso granular, respaldos verificados, y registros de auditoría de acceso. El incumplimiento puede resultar en multas de hasta 100 UIT (Unidad Impositiva Tributaria).

🎯 Objetivos de Aprendizaje

  • 🎯 Identificar y comparar los modelos de tecnología de almacenamiento: OLTP, OLAP, NoSQL, NewSQL, Data Lake, Data Lakehouse.
  • 🎯 Diseñar estrategias de backup y recuperación ante desastres (BCP/DRP) para datos críticos.
  • 🎯 Aplicar técnicas de optimización: indexación, particionamiento, compresión y caching.
  • 🎯 Implementar controles de seguridad técnica en el almacenamiento según Ley 29733.
  • 🎯 Evaluar criterios de selección de plataformas de almacenamiento on-premise vs. cloud.

🔑 Conceptos Clave

  • 🔑 OLTP (Online Transaction Processing): Sistemas optimizados para transacciones frecuentes y concurrentes: bancos de datos operacionales.
  • 🔑 OLAP (Online Analytical Processing): Sistemas optimizados para consultas analíticas complejas sobre grandes volúmenes de datos históricos.
  • 🔑 Data Warehouse: Repositorio centralizado de datos estructurados, históricos e integrados para análisis empresarial.
  • 🔑 Data Lake: Repositorio de datos en formato nativo (estructurado, semiestructurado, no estructurado) a bajo costo.
  • 🔑 Data Lakehouse: Arquitectura híbrida que combina la flexibilidad del Data Lake con la calidad y gobierno del Data Warehouse.

⚙️ Explicación Técnica Detallada

La gestión del Almacenamiento y Operaciones de Datos cubre el ciclo de vida completo:

  • 🚀 1. Aprovisionamiento: Selección de tecnología, instalación y configuración del motor de base de datos.
  • ⚙️ 2. Operación: Monitoreo de rendimiento, gestión de espacio y mantenimiento de índices.
  • 🛡️ 3. Respaldo y Recuperación (BCP/DRP): Políticas de backup (completo, incremental, diferencial) y pruebas de restore periódico.
  • 🗄️ 4. Archivado y Eliminación: Políticas de retención alineadas con requisitos legales (Ley 29733 y normativas sectoriales).
  • ⚡ 5. Optimización: Tuning de consultas, particionamiento, sharding y caching.

La arquitectura de almacenamiento analítico moderna suele seguir patrones escalables como la Arquitectura Lambda (Batch + Streaming) o Kappa (Solo Streaming).

🏢 Ejemplos en Organizaciones

Claro Perú migró sus datos de facturación (más de 15 millones de registros mensuales) de Oracle on-premise a Amazon Redshift, logrando una reducción del 65% en costos de almacenamiento y mejorando los tiempos de consulta analítica de 4 horas a 8 minutos. Implementaron cifrado AES-256 en reposo para cumplir con la Ley 29733 en la protección de datos de clientes.

✅ Buenas Prácticas Internacionales

  • ✔️ Diseñar la estrategia de almacenamiento alineada con el ciclo de vida del dato (activo, inactivo, archivado).
  • ✔️ Implementar siempre cifrado en reposo (AES-256) para datos personales y confidenciales.
  • ✔️ Probar los procesos de restauración de respaldos al menos mensualmente.
  • ✔️ Documentar las políticas de retención y eliminación de datos alineadas con requisitos legales.
  • ✔️ Usar particionamiento estratégico para mejorar el rendimiento de consultas analíticas frecuentes.

⚠️ Errores Comunes

  • ❌ Almacenar indefinidamente todos los datos sin políticas de retención ni eliminación.
  • ❌ No cifrar datos personales en reposo, incumpliendo el artículo 9 de la Ley 29733.
  • ❌ No probar los backups regularmente hasta que se necesita una recuperación de emergencia.
  • ❌ Elegir tecnologías de moda en lugar de las adecuadas para los casos de uso reales.
  • ❌ No planificar la capacidad de almacenamiento, generando crisis operativas por espacio insuficiente.

💼 Casos de Uso Reales

Amazon usa más de 100 servicios de almacenamiento distintos en su infraestructura AWS, incluyendo S3, DynamoDB, RDS y Redshift, eligiendo el motor adecuado para cada patrón de acceso. En Perú, el MEF implementó un Data Warehouse con SQL Server Analysis Services para el Presupuesto por Resultados, procesando datos de más de 2,000 entidades públicas con reportes en tiempo real para el Congreso.

🛠️ Herramientas y Tecnologías

  • Amazon Redshift: Data Warehouse columnar en la nube, ideal para analítica a gran escala.
  • Snowflake: Data Warehouse as a Service con separación de almacenamiento y cómputo, compatible con DAMA.
  • Apache Iceberg / Delta Lake: Formatos de tabla open-source para Data Lakehouses con soporte ACID.
  • Oracle Database / PostgreSQL: Motores OLTP robustos para datos transaccionales críticos.
  • AWS S3 / Azure Blob Storage: Almacenamiento de objetos económico para Data Lakes empresariales.

✏️ Ejercicio Práctico

Actividad: Diseña la arquitectura de almacenamiento para una empresa de retail peruana con 200 tiendas, 5 millones de transacciones diarias y requisito de analítica en tiempo real. Especifica: (a) Motores de almacenamiento por capa (transaccional, staging, analítica, archivado); (b) Política de retención de datos de clientes según Ley 29733; (c) Estrategia de backup con RTO y RPO; (d) Controles de seguridad técnica para datos personales; (e) Estimación de costos on-premise vs. cloud (Redshift/Snowflake).

💡 Recomendaciones de Implementación

  • 📌 Comenzar por el inventario de datos existentes y sus patrones de acceso antes de elegir tecnología.
  • 📌 Diseñar la política de retención considerando obligaciones contractuales, fiscales y de la Ley 29733.
  • 📌 Implementar monitoreo automatizado de rendimiento y alertas antes de que los problemas afecten a usuarios.
  • 📌 Adoptar el cifrado como estándar desde el inicio, no como medida correctiva posterior.
  • 📌 Evaluar la arquitectura Data Lakehouse para organizaciones que requieren tanto flexibilidad como gobierno.

📌 Resumen del Módulo

El Almacenamiento de Datos es la infraestructura operativa fundamental del ecosistema de información. El DAMA-DMBOK lo enmarca como un dominio de gestión que requiere tanto competencia técnica (tuning, backup, disponibilidad) como alineación con el gobierno y el cumplimiento. En el contexto peruano, la Ley 29733 impone requisitos técnicos concretos de seguridad en el almacenamiento de datos personales que convierten este dominio en una responsabilidad legal de primer orden.
