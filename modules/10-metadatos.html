📋 Descripción del Módulo

Los Metadatos son "datos sobre los datos": información que describe, explica y facilita la localización, el uso y la gestión de otros datos. La Gestión de Metadatos es el proceso de inventariar, clasificar, publicar y mantener este conocimiento para hacer los datos comprensibles, confiables y accesibles en la organización.

📘 Relación con DAMA-DMBOK

En el DAMA-DMBOK, la Gestión de Metadatos es el dominio que provee el contexto necesario para que todos los demás dominios funcionen efectivamente. Define tres tipos de metadatos: (1) Business — definiciones de negocio, glosarios, Data Owners; (2) Technical — estructuras de datos, modelos, formatos, APIs; (3) Operational — estadísticas de uso, linaje de datos, registros de calidad. El catálogo de datos es la herramienta central de este dominio.

⚖️ Relación con Gobierno Corporativo

Desde el gobierno corporativo, la Gestión de Metadatos es la base del principio de transparencia informacional. El directorio y los reguladores deben poder conocer qué datos tiene la organización, dónde están, quién los gestiona y cómo se usan. Los reguladores como la SBS y la SMV exigen que las entidades financieras mantengan catálogos de datos comprensibles para auditorías.

🔒 Cumplimiento — Ley N.° 29733

La Ley 29733 exige que las organizaciones registren sus bancos de datos personales ante la ANPDP, describiendo su contenido, finalidad, responsable y medidas de seguridad. Este registro es esencialmente un inventario de metadatos de bancos de datos personales. Un catálogo de datos bien mantenido facilita enormemente este cumplimiento y la capacidad de responder solicitudes de acceso, rectificación y cancelación (derechos ARCO).

🎯 Objetivos de Aprendizaje

  • 🎯 Comprender los tres tipos de metadatos del DAMA-DMBOK: business, técnicos y operacionales.
  • 🎯 Diseñar e implementar un Glosario de Negocio (Business Glossary) como primer producto de valor.
  • 🎯 Construir un Catálogo de Datos corporativo con lineaje, propietarios y clasificación.
  • 🎯 Gestionar el linaje de datos (Data Lineage) para garantizar trazabilidad end-to-end.
  • 🎯 Cumplir el requisito de registro de bancos de datos ante ANPDP usando el catálogo de metadatos.

🔑 Conceptos Clave

  • 🔑 Metadatos de Negocio: Definiciones, reglas de negocio, glosarios y documentación orientada a usuarios no técnicos.
  • 🔑 Metadatos Técnicos: Esquemas, modelos de datos, formatos de archivo, APIs y especificaciones de integración.
  • 🔑 Metadatos Operacionales: Estadísticas de uso, registros de calidad, tiempos de carga, perfiles de datos automáticos.
  • 🔑 Linaje de Datos: Registro de la trayectoria de un dato desde su origen hasta su destino final, incluyendo todas las transformaciones.
  • 🔑 Catálogo de Datos: Repositorio central de metadatos que permite a los usuarios descubrir, entender y acceder a los datos disponibles.

⚙️ Explicación Técnica Detallada

La gestión de Metadatos (el "contexto sobre los datos") avanza a través de 4 niveles de madurez funcional:

  • 📜 1. Inventario: Registro básico, usualmente manual, de sistemas, entidades y elementos técnicos críticos.
  • 📖 2. Glosario de Negocio: Unificación de definiciones organizacionales formales aprobadas por los Data Owners (el "mismo idioma" para todos).
  • 🔍 3. Catálogo de Datos: Plataforma central tecnológica que unifica metadatos técnicos (esquemas) y operacionales (volúmenes, horarios) con análisis avanzado de linaje.
  • 🧠 4. Red de Conocimiento (Active Metadata): Catálogos impulsados por Machine Learning que aplican clasificaciones automáticas, detección de PII y alertas autónomas de anomalías.

🏢 Ejemplos en Organizaciones

BBVA implementó un catálogo de datos empresarial con más de 120,000 activos de datos catalogados en 12 países. La búsqueda de datos por analistas que antes tomaba 3 días pasó a 15 minutos. Además, el catálogo sirve como evidencia de cumplimiento ante reguladores europeos (GDPR) e identificación automática de datos personales para la gestión de derechos ARCO. En Perú, el MIDIS usa un catálogo para gestionar los datos del Padrón General de Hogares con más de 8 millones de beneficiarios.

✅ Buenas Prácticas Internacionales

  • ✔️ Publicar el glosario de negocio como primer entregable visible del programa de metadatos.
  • ✔️ Automatizar la recolección de metadatos técnicos con herramientas de perfilado automático.
  • ✔️ Mantener el linaje de datos actualizado como parte del proceso de desarrollo y despliegue.
  • ✔️ Asignar responsables (Metadata Stewards) por dominio de datos para mantener los metadatos actualizados.
  • ✔️ Usar el catálogo como la fuente centralizada para el registro de bancos de datos personales ante ANPDP.

⚠️ Errores Comunes

  • ❌ Crear glosarios que nadie usa porque no están integrados con las herramientas de trabajo cotidiano.
  • ❌ No mantener los metadatos actualizados, generando un catálogo obsoleto que pierde credibilidad.
  • ❌ Documentar solo los metadatos técnicos ignorando los de negocio, que son los más valiosos para los usuarios.
  • ❌ No establecer un proceso de aprobación para las definiciones del glosario, generando versiones inconsistentes.
  • ❌ No documentar los metadatos de datos personales, dificultando el cumplimiento de la Ley 29733.

💼 Casos de Uso Reales

Google implementó un sistema interno llamado "Goods" que automáticamente catalogó más de 26 millones de tablas en su infraestructura de datos, con descubrimiento basado en machine learning. Este sistema eliminó miles de horas anuales de búsqueda de datos por parte de ingenieros y analistas. En el sector público latinoamericano, el gobierno de Uruguay implementó el primer catálogo de datos abiertos de América Latina con más de 1,200 activos de datos gubernamentales públicamente accesibles.

🛠️ Herramientas y Tecnologías

  • Collibra Data Catalog: Plataforma enterprise de catálogo, glosario y linaje con gobernanza integrada.
  • Microsoft Purview: Catálogo de datos Azure con descubrimiento automático y clasificación de datos personales.
  • Atlan: Catálogo de datos moderno con colaboración social, lineaje automático y integración con dbt.
  • Apache Atlas: Catálogo open-source con linaje para ecosistemas Hadoop y Spark.
  • DataHub (LinkedIn): Plataforma open-source de metadatos con linaje automático y SDK para integraciones custom.

📚 Frameworks y Estándares de Referencia

📐
ISO/IEC 11179 · ISO/IEC

Estándar internacional para registros de metadatos: define cómo describir, nombrar y registrar elementos de datos de manera consistente.

🏷️
Dublin Core · DCMI (Dublin Core Metadata Initiative)

Conjunto de 15 elementos de metadatos estandarizados para describir recursos digitales, ampliamente adoptado en bibliotecas y catálogos.

🔄
Common Warehouse Metamodel (CWM) · OMG (Object Management Group)

Especificación de metamodelo para intercambio de metadatos entre herramientas de Data Warehouse y Business Intelligence.

🌐
W3C DCAT · World Wide Web Consortium

Data Catalog Vocabulary: vocabulario RDF para describir catálogos de datos, facilitando la interoperabilidad entre catálogos de datos abiertos.

✏️ Ejercicio Práctico

Escenario: Minera Cobre Sur S.A. opera 2 minas de tajo abierto en Tacna con 15 sistemas de información (ERP SAP, MES, SCADA, DWH, CRM, RRHH, Geología, Medio Ambiente, Seguridad, Mantenimiento, Logística, Finanzas, Legal, Calidad y BI). Nadie en la empresa sabe exactamente qué datos hay en cada sistema ni cómo se relacionan. El directorio necesita la métrica "Costo por Tonelada de Cobre" pero 3 áreas la calculan diferente. Se decide implementar un catálogo de datos corporativo.

Entregables:

  • 1. Plan de inventario de datos: metodología para catalogar los 15 sistemas en 8 semanas. Define: qué metadatos técnicos recopilar (tablas, columnas, tipos, volumen), qué metadatos de negocio (definición, owner, clasificación), herramienta propuesta (Apache Atlas, Collibra, Datahub) y cronograma por sistema.
  • 2. Glosario de negocio minero: 12 términos críticos con: nombre estándar, definición de negocio aprobada, sinónimos usados en cada sistema, Data Owner, Data Steward y regla de cálculo si aplica. Alineado con ISO 11179 para registro de elementos de datos.
  • 3. Lineaje end-to-end de la métrica "Costo por Tonelada de Cobre": diagrama desde las 4 fuentes (SAP costos, MES producción, SCADA consumo energético, Logística transporte) pasando por las transformaciones del DWH hasta el dashboard del directorio. Identifica cada transformación y regla de cálculo.
  • 4. Inventario de bancos de datos personales: tabla con cada sistema que almacena datos personales de empleados, contratistas o comunidades aledañas. Incluye: tipo de dato personal, cantidad de titulares, base legal, finalidad, y estado de registro ante la ANPDP. Referencia: W3C DCAT para descripción de datasets.
  • 5. Modelo de gobierno del catálogo: roles (Administrador del catálogo, Data Stewards por dominio, consumidores), proceso de solicitud de nuevos términos, flujo de aprobación de definiciones, frecuencia de revisión y métricas de salud del catálogo (% de cobertura, % de términos con owner asignado).

Formato de entrega: Plan de inventario + glosario en tabla + diagrama de lineaje + inventario de datos personales + modelo de gobierno. Marcos sugeridos: ISO 11179, Dublin Core, CWM, W3C DCAT.

💡 Recomendaciones de Implementación

  • 📌 Integrar el catálogo de datos con las herramientas de trabajo cotidiano (Slack, Teams, IDE) para maximizar adopción.
  • 📌 Comenzar con los datos más críticos y con mayor demanda de los usuarios, no intentar catalogar todo.
  • 📌 Establecer un proceso automatizado de perfilado de datos para mantener metadatos operacionales actualizados.
  • 📌 Vincular el catálogo de datos con el programa de Gobierno de Datos: mismos Data Owners, mismas políticas.
  • 📌 Usar el catálogo como la fuente para cumplir con el artículo 14 de la Ley 29733: informar a los titulares sobre sus datos.

📌 Resumen del Módulo

La Gestión de Metadatos es el dominio del DAMA-DMBOK que hace los datos comprensibles, descubribles y confiables. Un catálogo de datos bien implementado es la columna vertebral del programa de Gobierno de Datos y el instrumento que facilita tanto la productividad de los equipos de datos como el cumplimiento regulatorio. En el contexto peruano, el catálogo de datos es también el instrumento más eficiente para cumplir con las obligaciones de registro y transparencia de la Ley 29733 ante la ANPDP.
